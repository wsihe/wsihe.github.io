Vue2.0多页签应用(8)-组件通信

2017.07.31

ypa框架中页面通过框架提供的几种方式实现页面间通信，其中事件广播：

  • 事件方法

事件方式适用于监听页面外部模块触发的动作，该动作对当前页面的数据和展示造成一定的影响，比如组合详情页面监听组合标题修改事件：

  • 监听事件

  • 发送事件

这种模式类似一个“单向数据流”理念：

  • state，驱动应用的数据源；
  • view，以声明方式将state映射到视图；
  • actions，响应在view上的用户输入导致的状态变化

当我们的应用遇到多个组件共享状态时，单向数据流的简洁性很容易被破坏：

  • 多个视图依赖于同一状态。
  • 来自不同视图的行为需要变更同一状态。

所以这个时候，我们需要用Vuex负责多组件的状态管理。

Vuex 使用单一状态树，我们可以把理解成一数据的集合，相当于一个“前端数据库”，让其在各个页面上实现数据的共享，并且可操作。

这意味着本来需要共享状态的更新是需要组件之间通讯的，而现在有了vuex，就组件就都和store通讯了，问题就自然解决了
