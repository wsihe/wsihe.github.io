前端工程化实践

2017.10.01

在开发 Wuui 组件库时前端工程化的实践, 什么是前端工程化呢，我个人想法前端工程化分为模块化／组件化／规范化／自动化，是制定统一的标准，提高效率。

模块化

模块化是将一个大文件拆分成多个相互依赖的小文件，再进行统一的拼装在一起。
模块化开发一般分为 JS的模块化，CSS模块化，资源模块化

  • JS模块化：AMD/CMD，CommonJs的加载规范和ES6的模块系统
  • CSS模块化：CSS in JS，CSS Module
  • 资源模块化：所有资源都能进行模块化

组件化

  • 页面上的每个 独立的 可视/可交互区域视为一个组件;
  • 每个组件对应一个工程目录，组件所需的各种资源都在这个目录下就近维护;
  • 每个组件相对独立，页面只不过是组件的容器，组件自由组合形成功能完整的界面;
  • 当不需要某个组件，或者想要替换组件时，可以整个目录删除/替换。

总的来说：从UI拆分下来的每个包含模板(HTML)+样式(CSS)+逻辑(JS)功能完备的结构单元，我们称之为组件

自动化部署

使用Travis CI自动部署到 Github 上，Travis CI 是目前新兴的开源持续集成构建项目，它与jenkins，GO的很明显的特别在于采用yaml格式，同时他是在在线的服务，不像jenkins需要你本地打架服务器，简洁清新独树一帜。目前大多数的github项目都已经移入到Travis CI的构建队列中，据说Travis CI每天运行超过4000次完整构建。
