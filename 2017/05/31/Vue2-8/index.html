组件通信 (vuex)

2017.05.31

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化，这是官方的说明。

vue中的 global event bus

在简单的场景下，可以使用一个空的 Vue 实例作为中央事件总线

var bus = new Vue()

// 触发组件 A 中的事件
bus.$emit(‘id-selected’, 1)

// 在组件 B 创建的钩子中监听事件
bus.$on(‘id-selected’, function (id) {
// …
})

但是当我们的应用遇到多个组件共享状态时，这样的global event bus很容易被破坏：

  • 多个视图依赖于同一状态。
  • 来自不同视图的行为需要变更同一状态。

所以这个时候，我们需要用Vuex负责多组件的状态管理。

Vuex

Vuex 使用单一状态树，我们可以把理解成一数据的集合，相当于一个“前端数据库”，让其在各个页面上实现数据的共享，并且可操作。

这意味着本来需要共享状态的更新是需要组件之间通讯的，而现在有了vuex，就组件就都和store通讯了，问题就自然解决了
