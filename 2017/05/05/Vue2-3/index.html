Vue-cli Webpack 配置分析 (dev)

2017.05.05

vue-cli 是 vuejs 官方提供的基于 vuejs 的项目脚手架工具, 可以很快的帮助 vuejs 开发者搭建一个 startup 项目, 免去环境配置的繁琐, 开箱即用. 今天就来看下 vue-cli 在开发环境（dev）的实现

安装

  • 安装前检查npm版本，要求3.10以上，npm下载地址
    $ npm -v
# 如果是旧版，请升级
$ npm upgrade -g

  • npm安装很慢（国外服务器），可以关联淘宝源

    $ npm install -g cnpm --registry=https://registry.npm.taobao.og

  • 安装vue-cli，创建项目

    # 安装 webpack
$ npm install webpack --global
# 全局安装 vue-cli
$ npm install --global vue-cli
# 创建一个基于 webpack 模板的新项目
$ vue init webpack asset2
# 安装依赖，走你
$ cd asset2
$ npm install
$ npm run dev

这里有个要注意的地方，要全局安装webpack，否则会安装失败，安装过程会询问是否需要添加一些相关的依赖（编码规范、测试等）

目录结构

关系

执行 npm run dev 的时候，引入的关系如下：
npm run dev => dev-server.js => webpack.dev.conf.js(index.js) => webpack.base.conf.js

dev-server.js

最先执行的是 dev-server.js 主要做了以下事情

  • 引入相关配置
  • 创建express 服务
  • webpack 编译
  • 配置开发中间件和热重载中间件
  • 代理服务器- 自动打开浏览器

代码注释

webpack.dev.conf

开发环境环境下配置，主要做了一下事情

  • 合并 webapck.base.conf.js
  • 使用 styleLoaders
  • 配置 Source Maps
  • 配置 webpack 插件
  • 将 hot-reload 相关代码添加到 entry chunks

代码注释

webpack.base.conf.js

基础配置，主要做了以下事情

  • 配置编译入口 / 出口
  • 配置不同类型模块的处理规则

代码注释

config / index.js

config 文件夹下面最主要的文件就是index.js。主要做了开发环境和生产环境的配置。

代码注释

.babelrc

Babel解释器的配置文件，存放在根目录下。Babel是一个转码器，项目里需要用它将ES6代码转为ES5代码。这里有一篇阮一峰老师写的相关文章供参考：Babel 入门教程

.editorconfig

该文件定义项目的编码规范，编辑器的行为会与.editorconfig 文件中定义的一致，并且其优先级比编辑器自身的设置要高，这在多人合作开发项目时十分有用而且必要
