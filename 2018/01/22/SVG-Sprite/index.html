SVG Sprite 技术实践

2018.01.22

前端 icon 的发展：image/CSS Sprite => Icon Font => SVG Sprite

背景

在项目中我们经常会遇到使用多个icon的情况，为了减少网页的http请求，提高页面的性能，通常的做是会把icon合并到一个文件中，再使用相应的技术把所需的图形从icon集合文件中取出并显示。目前使用的比较多的应该就是 CSS Sprite和 Icon Font。

CSS Sprite

CSS Sprite 在国内很多人叫css精灵，其原理是将需要分别显示的多张图像集成为单一图像，然后利用层叠样式表分别定位显示各部分图的技术，以减少下载图像数量，提高网页显示速度，background-position可以用数字精确的定位出背景图片的位置。

实现

1.在旧的项目，可能大多的实现通过 photoShop 来生成雪碧图， 并且雪碧图以PSD文件格式保留， 后续修改图标（或新增图标）只需要找到并修改该PSD。

2.现代的前端开发流程，我们可以使用前端构建工具来实现雪碧图，例如 Gulp / webpack

1.优点

减少HTTP请求，增加图片信息重复度，提高压缩比，减少图片大小，更换风格方便，只需在一张或几张图片上修改颜色或样式即可实现

2.缺点

不同颜色的icon要各保存一份，icon放大会有锯齿，往往要生成多个图片

Icon Font

Icon Font 将矢量的图标做成字体，把所有的图标都放在一个字体里面，就是一个字体库了，然后通过 @font-face 来引入。

1.优点

轻量（文件体积小）、灵活（样式可改变图标）、兼容性好（IE8+）。

2.缺点

图标只能单色调（太复杂的多色图标无法实现）、生成图标字体相对花时间，创作自已的字体图标很费时间，后期维护的成本偏高

SVG Sprite

SVG Sprite 类似于CSS中的Sprite技术，图标图形整合在一起，实际呈现的时候准确显示特定图标，更详细的介绍可以查看张鑫旭大大的文章未来必热：SVG Sprite技术介绍

生成 svg-sprite

怎么优雅的生成 svg-sprite呢？ 可以使用 webpack loader 来生成并导入html中，例如 svg-inline-loader 或者 svg-sprite-loader

在vue-cli的基础上，做一些改造才能引入 svg-inline-loader，要不会造成冲突

通过webpack的include对指定的svg文件夹进行处理

一般来说 ，svg文件会包含部分冗余的信息，如注释 <!-- Generator: Sketch 47.1 (45422) - http://www.xxxx.com/sketch -->

可以使用 svgo-loader 对 svg 优化处理

使用 icon 组件

我们有了 svg-sprite 图标，接下来就是如何在自己的项目中优雅的使用它

1.优点

支持多色图标了，不再受单色限制。
支持像字体那样通过font-size,color来调整样式。
可利用CSS实现动画。
减少HTTP请求。
矢量，缩放不失真
可以很精细的控制SVG图标的每一部分

2.缺点

SVG兼容性支持ie9+, 其渲染性能较差
